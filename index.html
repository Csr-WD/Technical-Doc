Introduction

JavaScript is a versatile programming language that runs in the browser and on servers (via Node.js). In a web page you add it with a <script> tag to make content interactive and dynamic.

Modern sites load scripts without blocking rendering using defer or type="module". Verify execution quickly via DevTools Console with console.log().

Basic Syntax

Use let and const to declare variables; prefer const for bindings you won’t reassign. Variables are block-scoped inside { }, avoiding leaks to outer scopes.

Statements may end with a semicolon ; (ASI exists, but being explicit is safer). Single-line comments start with //, and multi-line comments use /* ... */.

Data Types

JavaScript has primitive and reference types. The typeof operator inspects values: for example, typeof 42 === "number" and typeof "hello" === "string". Objects and arrays are reference types (assigned and compared by reference).

Two special values are undefined (declared but not assigned) and null (explicit “no value”). Historical quirk: typeof null === "object".

  • Number – numeric values like 3.14 or -2
  • String – text like "hello"
  • Boolean – true or false
  • Null – explicit empty value: null
  • Undefined – unassigned value: undefined

Functions

A function is a reusable block of code. You can define one with a declaration or an expression. Declarations are hoisted (callable before they appear), expressions are not. Use function and return to define behavior and produce a value.

Arrow functions use the => syntax and have lexical this. They’re concise and support implicit returns for single expressions. Prefer them for short callbacks; keep declarations for named, reusable APIs.

function greet(name) {
  return 'Hi ' + name;
}

const double = (n) => n * 2;

DOM and Events

The DOM is a tree that represents your HTML. Select elements with document.querySelector() and update properties like textContent, className, or attributes via setAttribute().

Events let the page react to user actions. Use addEventListener() with events like click, input, or keydown. Call event.preventDefault() to stop default behavior; use event delegation for dynamic UIs.

const btn = document.querySelector('#demo-btn');
const output = document.querySelector('#demo-out');

btn.addEventListener('click', () => {
  output.textContent = 'Button clicked!';
});
